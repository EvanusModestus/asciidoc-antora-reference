= AsciiDoc + Antora Reference
:description: A live, a-la-carte reference of every AsciiDoc and Antora feature, each shown as copyable source next to its rendered result.
:navtitle: Overview

A single, browsable reference that demonstrates *what AsciiDoc and Antora can do* — each feature shown as copyable source right next to the result the site produces from it.

Use it as an *a la carte menu*: skim the categories, find the effect you want, copy the syntax.

[TIP]
====
Every example on this site is *real*. The gray "`syntax`" box is the exact AsciiDoc that produced the rendered block immediately below it. Nothing is faked with screenshots.
====

== What each badge means

Each feature is tagged so you know where it works before you rely on it:

[cols="1,3a",grid=rows,frame=none]
|===
| [.compat.b-both]#AsciiDoc + Antora# | Core AsciiDoc syntax that renders the same whether processed by Asciidoctor directly or through the Antora pipeline. Safe everywhere.
| [.compat.b-adoc]#AsciiDoc only# | Standard AsciiDoc/Asciidoctor behavior that Antora intentionally changes, restricts, or discourages (for example, filesystem includes or `toc::[]`). Works in plain Asciidoctor; read the note before using it in Antora.
| [.compat.b-antora]#Antora only# | Provided by the Antora pipeline, not the AsciiDoc language itself — resource IDs, cross-component xrefs, page attributes, navigation files, and friends. These do nothing in a bare Asciidoctor run.
|===

See xref:compatibility-legend.adoc[] for the full rundown, including the handful of features that behave differently in each tool.
